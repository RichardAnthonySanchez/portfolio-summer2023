project difficulties

the first iteration of this project displayed a spread of ten cards with information attached to each card. this was daunting for new users and wasn't practical.

my solution

now the display on start up shows one card at a time with its a prompt. there's a text area for the user's entry, where they reflect on their experiences. one prompt per card allowed users to associate and learn more efficiently and the project meant a lot more to people when it became their journal instead of just a database.

project difficulties

when i first made masterpiece hub, i had mainly planned through the user experience. this caused me to focus on unnecessary features instead of the utility of the project.

my solution

i removed code that allowed users to search artworks by name. this would go largely unused. our target users are using this platform to educate themselves. a search bar would only apply to people who are already familiar with artworks, so deleting the feature was necessary. this allowed me to make time to automate the generation of the artwork pages - a feature that saved countless hours.
